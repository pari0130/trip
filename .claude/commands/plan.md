# /plan - 구현 계획 수립

$ARGUMENTS 에 대한 구현 계획을 수립합니다.

## 워크플로우

1. **요구사항 분석**: $ARGUMENTS 의 목적과 범위 파악
2. **기존 코드 탐색**: 영향받는 파일과 컴포넌트 파악
3. **패턴 확인**: 프로젝트 컨벤션 및 재사용 가능한 패턴 확인
4. **계획 작성**: 아래 형식으로 구현 계획 출력

## 출력 형식

```markdown
# 구현 계획: [기능명]

## 개요
[2-3문장 요약]

## 영향 범위
- 수정: [파일 목록]
- 생성: [파일 목록]

## 구현 단계

### Phase 1: [단계명]
1. **[작업명]** (File: path/to/file.kt)
   - 작업: 구체적 내용
   - 이유: 필요한 이유
   - 위험: 낮음/중간/높음

## 테스트 전략
- Unit: [대상]
- Integration: [시나리오]

## 완료 기준
- [ ] 기준 1
- [ ] 기준 2
```

## 규칙
- 코드 작성 금지, 계획만 수립
- 정확한 파일 경로와 함수명 사용
- 동시성 영향 반드시 평가
- 기존 패턴 유지 우선
